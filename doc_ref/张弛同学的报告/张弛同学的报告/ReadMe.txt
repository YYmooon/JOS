同学们好，这份报告是我在大三学期选修操作系统实习JOS版本的完整课程报告，这份报告
是我在阅读MIT的课程指导以及华中课大学邵志远老师之后结合自己在实习过程中遇到得问题
对于上述两份资料的补充和一点个人得心得。

整个作JOS的过程中我是一边写报告一边写代码的，任何我在写代码的时候遇到了问题，
我都会先将问题写进报告，记录下我当时的思路和产生问题的原因，写完代码调试完成以
后，再回来将解决的办法说明。因为写代码的时候很多情况下问题都是一个依赖另一个，
所以作程序的时候有一种感觉就是脑子里要压栈，一个解决以后再回到上一个依赖他的，
直到最后全部解决。但是这些东西如果到最后做完了才写，过程就一扫而空了，感觉东西
全在脑子里，就是倒不出来要写的东西。

就像我前面说得，报告写得时候主要不是为别人，而是为记录自己的收获和体会。如果只是
按照别人的思路将实习过程走一遍，那么体会仍然是别人的，疑问还是留在了自己得脑子里。
况且这份报告事后我也发现存在大量的错误。我建议大家比较好的方式是，读完MIT的资料后
开始做实习，实习做到出错以后带着自己的问题再阅读报告。这样当你的疑问和别人得想法发生
碰撞以后，你会感觉你的思考会比仅仅看报告要深入，也会有更多的收获可以写进自己的报告里
